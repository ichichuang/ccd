# UI Validation

Review the affected UI against these production criteria:

1. Architecture and component ownership remain valid.
2. Semantic tokens and existing components are reused.
3. Layout and scroll ownership are explicit at supported breakpoints.
4. Loading, empty, error, disabled, and success states remain coherent.
5. Keyboard order, focus visibility, labels, contrast, and reduced motion remain accessible.
6. Light and dark themes remain legible.
7. Type checking, linting, architecture checks, and affected app builds pass.
8. The final diff contains no unrelated changes or stale paths.
