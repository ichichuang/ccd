# Tokens and UnoCSS

Tokens and UnoCSS governance controls visual constants, semantic shortcuts, structural CSS, PrimeVue theme boundaries, and shared primitive dependencies. Agents must use registered ownership instead of local product styling constants.

## Ownership Layers

Token ownership flows from packages/design-tokens through theme engine, registered CSS variables, UnoCSS semantic shortcuts, PrimeVue theme configuration, and approved shared primitives.

This section requires agents to apply design token definitions rules with explicit ownership, current monorepo evidence, accessibility-safe behavior, token-driven styling, and validation before reporting completion.
