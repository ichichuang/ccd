# Interaction and Motion

Interaction and motion governance applies only when feedback, transitions, gestures, or animation tools are in scope. Agents must choose the smallest eligible tool, preserve reduced-motion behavior, and avoid default loading of motion systems.

## Scope

## Motion Tool Selection

Choose the smallest tool that satisfies the interaction. CSS or existing utilities come first; animate-lite is conditional for lightweight transitions; GSAP is conditional for timeline-heavy work.

## GSAP Eligibility

GSAP is eligible only for timeline sequencing, complex choreography, scroll-linked animation, or cases where existing utilities cannot express behavior. Ownership, cleanup, and reduced-motion fallback are required.

## Animate-Lite Eligibility

animate-lite is eligible only for lightweight route, class, enter or leave, or state transitions where existing utilities are insufficient. It is not a general motion default.

## Conditional Motion Loading

Do not load GSAP or animate-lite by default. Do not make both tools simultaneous defaults. Load only the needed motion reference and tool path.
