# Component Selection and Styling Boundaries

Component selection governance controls which existing owner must be used before new UI primitives are created. Agents must verify component exports, PrimeVue v4 APIs, adapter boundaries, states, accessibility, and runtime boundaries.

## Component Priority Order

Use this exact priority: 1. Existing `@ccd/vue-ui` component. 2. Existing `@ccd/vue-primevue-adapter` component or adapter. 3. Approved PrimeVue component through project boundaries. 4. Existing shared app component. 5. New shared primitive only after evidence proves no suitable owner exists.

## Existing CCD Vue UI Components

This section requires agents to apply existing ccd vue ui components rules with explicit ownership, current monorepo evidence, accessibility-safe behavior, token-driven styling, and validation before reporting completion.
