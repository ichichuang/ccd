## Login Diorama Refactor Plan — Card Breakout & Diorama

Target state:

- `apps/web-demo/src/views/login/index.vue` becomes a single integrated diorama panel instead of two independent left/right cards.
- Brand identity floats above the panel center; theme and locale controls remain in the viewport top-right.
- The login form occupies the left 55%–60% of the desktop panel and keeps `ProForm` as the only multi-field form boundary.
- `AnimatedCharacters` occupies the right 40%–45% as a physical scene object, aligned to the panel floor and breaking above the panel top by 15%–20%.
- Password field visual structure is repaired so prefix icon, input body, and mask toggle stay inside one aligned control shell.
- Light/dark theme, size density, semantic tokens, responsive breakpoints, and design-engine UnoCSS laws remain authoritative.

- [⬜️] [Login-P1-Composition] Remove duplicate brand placement inside the old visual panel and move brand/logo/title into a centered floating block above the diorama panel.
- [⬜️] [Login-P1-Password] Fix password icon misalignment caused by the separate absolute leading icon plus PrimeVue `Password` internal structure.
- [⬜️] [Login-P1-Depth] Improve dark-mode elevation using border plus inset-highlight semantics instead of relying on outer shadows alone.
- [⬜️] [Login-P1-VisualNoise] Reduce decorative copy/pills inside the main panel so the form and breakout character are the dominant hierarchy.

Desktop structure:

- [⬜️] [Login-P2-Shell] Create a centered horizontal panel with `width: min(92vw, 960px)` baseline, max expansion capped near `1200px`, and height governed by viewport-safe min/max values.
- [⬜️] [Login-P2-Grid] Define panel internal grid as `minmax(0, 58%) minmax(0, 42%)` on desktop and keep it non-nested with existing layout shortcut laws.
- [⬜️] [Login-P2-FormZone] Place login heading, subtitle, presets, fields, submit button, register link, and version footer in the left zone with consistent token gaps.
- [⬜️] [Login-P2-StageZone] Place the character stage in the right zone with `overflow: visible`, floor alignment, and no extra textual content.
- [⬜️] [Login-P2-Breakout] Position `AnimatedCharacters` so the body baseline aligns to the panel lower visual floor and the top breaks out above the panel by 15%–20%.
- [⬜️] [Login-P2-TopControls] Keep theme switch and locale select in a compact top-right toolbar with safe-area offsets.
- [⬜️] [Login-P2-BottomLinks] Keep help/privacy links centered below the panel and visually separated from the panel bottom.

Design-engine compliance:

- [⬜️] [Login-P5-Tokens] Use semantic color tokens only: `background`, `card`, `foreground`, `muted`, `border`, `primary`, `info`, `success`, `warn`, and related foreground/light variants.
- [⬜️] [Login-P5-Sizing] Use CSS variables, `%`, `vw`, and `vh` for authored sizing; avoid `rem`/`em` and avoid unnecessary raw px outside existing animation geometry.
- [⬜️] [Login-P5-Shortcuts] Use only registered shortcuts from `semanticShortcuts.ts`; do not invent login-specific UnoCSS shortcuts unless the design engine registry is intentionally extended.
- [⬜️] [Login-P5-Borders] Pair every border utility with explicit style and semantic color if authored in template classes.
- [⬜️] [Login-P5-ZIndex] Use `z-base`, `z-content`, `z-layout`, `z-overlay`, `z-popover`, or DOM stacking context restructuring only.
- [⬜️] [Login-P5-RuleOf7] Move complex layout and decorative geometry into scoped CSS classes when template utility count would exceed the Rule of 7.
- [⬜️] [Login-P5-Deep] Avoid adding new `:deep(.p-*)` selectors beyond the existing documented login exception unless a focused architectural justification is added.
