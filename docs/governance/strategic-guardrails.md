# Strategic Guardrails

This registry is the durable governance record for non-actionable P4 strategic guardrails. It closes the repair-ledger ambiguity without approving product work, repository splits, dependency additions, runtime promotions, desktop plugins, new Rust commands, or remote GitHub setting mutations.

The runtime repair ledger keeps these items visible to `pnpm ai:doctor --open` as warning-preserving guardrails only. The ledger is not an implementation backlog for these items.

## Policy

- Status `Deferred` means the item is not part of the current CCD architecture repair program and needs a future owner-approved lane.
- Status `Blocked` means current policy explicitly forbids the work unless a future owner decision replaces the block.
- Owner approval must record scope, allowed paths, forbidden paths, prerequisites, validation, rollback, and evidence before implementation begins.
- Future lanes must stay isolated. Do not mix strategic product work with governance cleanup, dependency modernization, runtime-boundary repair, or desktop security work.
- No item in this registry authorizes dependency upgrades, new Tauri plugins, new Rust commands, app-to-package runtime promotion, global `@ccd/*` aliases, organization/repository creation, starter extraction, or design-system split.

## Operating Rule

When a future request mentions any guardrail in this registry, treat it as blocked or deferred by default. The first valid step is an owner-decision update and scoped lane plan, not implementation.
