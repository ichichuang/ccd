# Example Route Inventory

This inventory documents the browser app-owned example/report route surface. It is descriptive only: route records, page components, app mock/demo data, app stores, and app adapters remain owned by `apps/web-demo`; reusable primitives consumed by those pages remain owned by their package boundaries.

## Source Of Truth

- Aggregator: `apps/web-demo/src/router/modules/example.ts`
- Split route modules: `apps/web-demo/src/router/modules/example/*.ts`
- Page roots: `apps/web-demo/src/views/example/**`
- Regression guard: `apps/web-demo/src/router/modules/example.spec.ts`

The route spec preserves the registered inventory, ordering, and split-module aggregation with `106` registered route records. Update this document when a route module or example page group is added, removed, or reassigned.

## Ownership Rules

- `apps/web-demo` owns route records, route metadata, page composition, app-local example data, app stores, browser adapters, and app integration shells.
- `packages/vue-ui` owns reusable CCD UI primitives such as ProForm, ProTable, CScrollbar, empty states, and CCD Prime controls.
- `packages/vue-primevue-adapter` owns PrimeVue runtime installation, services, locale, theme, and adapter facades.
- `packages/vue-charts` owns chart runtime helpers; chart example pages remain app-owned consumers.
- `packages/design-tokens`, `packages/unocss-preset`, `packages/vue-hooks`, and `packages/vue-app-platform` own reusable package primitives consumed by example pages.
- Example route/page code is not a public package export surface. Promote reusable behavior through governed packages only.
